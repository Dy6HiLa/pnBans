# pnBans

pnBans — плагин для модерации сервера Paper. Он позволяет выдавать баны, муты и предупреждения, смотреть историю нарушений и быстро разбираться с аккаунтами, которые могут принадлежать одному игроку.

## Что есть в плагине

### Наказания

Модератор может выдать бан, мут, предупреждение или бан по IP. У каждого наказания есть срок и причина.

```text
/ban Nick 30d Использование читов
/mute Nick 2h Флуд
/warn Nick 1d Оскорбления
/ipban Nick 7d Обход наказания
```

Можно настроить готовые правила с кодами. Тогда не нужно каждый раз писать срок и причину вручную:

```text
/ban Nick 1.1
/mute Nick 2.1
```

Скрытое наказание добавляется с `-s`. Обычные игроки не увидят сообщение в чате, но персонал и консоль получат уведомление.

```text
/ban Nick 30d Использование читов -s
```

### История игрока

Команда `/history Nick` открывает историю наказаний игрока. Там можно посмотреть, за что и когда он был наказан, кто выдал наказание и активно ли оно сейчас.

Также есть история действий персонала:

```text
/history staff
/history staff NickModeratora
```

## DupeIP — проверка связанных аккаунтов

DupeIP помогает модератору понять, есть ли у игрока другие аккаунты. При входе игрока система сравнивает его данные с уже известными аккаунтами и при подозрительной связи отправляет сообщение персоналу.

Проверить игрока вручную:

```text
/dupeip Nick
```

Проверка ищет не просто одинаковый IP. Она показывает, почему аккаунты связаны:

- один и тот же IP или несколько общих IP;
- похожие ники;
- недавнее появление нового аккаунта после бана;
- быструю смену аккаунтов на одном IP;
- активные баны и муты на связанном аккаунте.

Для каждой связи выводятся причина, уровень риска, известные IP, страна, время последнего входа и история наказаний. Из GUI можно сразу открыть историю нужного аккаунта.

DupeIP не банит автоматически. Он даёт модератору информацию, а решение всегда остаётся за человеком. Координаты игроков в проверке и уведомлениях не показываются.

### Уведомления для персонала

Когда появляется подозрительная связь, модераторы с правом `pnbans.dupeip.alert` получают уведомление с причиной и кнопкой для открытия `/dupeip`. Чтобы не было спама, одинаковые уведомления имеют настраиваемую задержку.

В конфиге можно выбрать, на что реагировать: совпадение IP, похожий ник или определённый уровень риска. Также можно добавить IP и игроков в исключения — они останутся в истории, но не будут создавать тревоги.

## Команды

| Команда | Что делает |
| --- | --- |
| `/ban <ник> <срок> <причина>` | Выдать бан. |
| `/unban <ник>` | Снять бан. |
| `/ipban <ник\|ip> <срок> <причина>` | Заблокировать IP. |
| `/unipban <ник\|ip>` | Снять IP-бан. |
| `/mute <ник> <срок> <причина>` | Выдать мут. |
| `/unmute <ник>` | Снять мут. |
| `/warn <ник> <срок> <причина>` | Выдать предупреждение. |
| `/unwarn <ник>` | Снять предупреждение. |
| `/history <ник>` | Открыть историю игрока. |
| `/history staff [модератор]` | Посмотреть действия персонала. |
| `/dupeip <ник>` | Проверить связанные аккаунты. |
| `/pnbans reload` | Перезагрузить настройки и сообщения. |

Сроки пишутся так: `30m`, `1h`, `1h30m`, `7d`, `2w`, `1mo`.

## Права

| Право | Для чего нужно |
| --- | --- |
| `pnbans.ban`, `pnbans.unban` | Баны. |
| `pnbans.ban.ip`, `pnbans.unban.ip` | IP-баны. |
| `pnbans.mute`, `pnbans.unmute` | Муты. |
| `pnbans.warn`, `pnbans.unwarn` | Предупреждения. |
| `pnbans.history` | Просмотр истории. |
| `pnbans.dupeip` | Ручная проверка DupeIP. |
| `pnbans.dupeip.alert` | Получение уведомлений DupeIP. |
| `pnbans.dupeip.bypass` | Не проверять игрока через DupeIP. |
| `pnbans.admin` | Перезагрузка плагина. |

## Установка

1. Поместите JAR-файл в папку `plugins` сервера.
2. Запустите сервер.
3. Настройте `plugins/pnBans/config.yml`: причины, сроки, права, уведомления и правила DupeIP.
4. При необходимости измените тексты в `messages.yml`.

После изменения настроек используйте `/pnbans reload`.
