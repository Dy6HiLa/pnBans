# pnBans

`pnBans` — система модерации для Paper 1.21+: баны, муты, предупреждения, аудит действий персонала, история наказаний и DupeIP для проверки связанных аккаунтов.

Главная идея DupeIP: не принимать совпадение IP за готовый вердикт. Система собирает несколько сигналов, объясняет их модератору и показывает риск в удобном GUI. Решение о наказании всегда остаётся за персоналом.

## Что умеет плагин

- Баны, IP-баны, муты и предупреждения с ограниченным сроком.
- Скрытые наказания через `-s`: обычные игроки не видят объявление, а персонал и консоль получают уведомление.
- Причины и готовые пресеты наказаний из `config.yml`, например `/mute Player 2.1`.
- Ограничения по группам Vault: cooldown и максимальный срок каждого типа наказания.
- История игрока и история действий модератора в GUI.
- SQLite, MySQL и MongoDB.
- Проверка обновлений через pnLibrary и анонимная статистика bStats.

## DupeIP: как работает проверка связей

При входе игрока DupeIP обновляет его профиль и асинхронно сравнивает его с известными аккаунтами. Ручная проверка открывается командой:

```text
/dupeip <ник>
```

DupeIP учитывает следующие сигналы:

| Сигнал | Что означает |
| --- | --- |
| Общий IP | Аккаунты использовали один и тот же IP-адрес. |
| Похожие ники | Сравнивается схожесть основы ника; порог задаётся в конфиге. |
| История IP | Учитываются не только текущие, но и ранее замеченные IP. |
| Активные наказания | Проверяются баны и муты связанного аккаунта и его известных IP. |
| Возможный обход бана | Новый аккаунт появился на общем IP вскоре после бана. |
| Быстрая смена | Аккаунты быстро переключались на одном IP. |
| Новый аккаунт | UUID впервые замечен недавно и уже имеет связь по IP или нику. |

Из сигналов формируется риск от `0` до `100` и уровень: низкий, средний, высокий или критический. Балл риска — это подсказка, а не автоматический бан.

### Что видит модератор

- GUI с текущим IP, страной, количеством известных IP и найденными связями.
- Для каждой связи: общие IP, схожесть ника, последний вход, причины совпадения, риск и статистика банов/мутов/предупреждений.
- ЛКМ по аккаунту открывает его полную историю наказаний.
- Staff-alert при входе срабатывает по порогу риска, общему IP или похожему нику — всё настраивается.
- Координаты игроков в DupeIP, GUI и уведомлениях не выводятся.

### GeoIP и приватность

При включённом `dupeip.geoip` публичный IP может быть отправлен в `https://ipwho.is`, чтобы показать страну. Локальные IP наружу не отправляются, а результаты кэшируются в памяти. Отключить это можно через `dupeip.geoip.enabled: false`.

DupeIP хранит данные профиля и историю IP, чтобы находить связи между входами. Перед использованием на публичном сервере добавьте это в свою политику конфиденциальности.

## Быстрые примеры

```text
/ban Player 30d Использование читов
/ban Player 1.1
/mute Player 2.1
/warn Player 1d Нарушение правил
/ipban Player 7d Обход наказания
/ban Player 30d Читы -s
/history Player
/history staff
/dupeip Player
```

Время поддерживает `s`, `m`, `h`, `d`, `w`, `mo`: `30m`, `1h30m`, `7d`, `100d`. Бессрочные наказания намеренно не поддерживаются.

## Настройка DupeIP

Все параметры находятся в `config.yml` → `dupeip`.

| Параметр | Назначение |
| --- | --- |
| `enabled` | Включает DupeIP. |
| `alert-permission` | Право на staff-alert. |
| `alert-console` | Дублировать ли alert в консоль. |
| `alert-cooldown-seconds` | Защита от повторного спама об одной паре аккаунтов. |
| `ignored-ips`, `ignored-players` | Исключения: сохраняются в истории, но не создают связи и alert. |
| `name-similarity-threshold` | Порог похожести ников, по умолчанию `0.82`. |
| `detection.alert-risk-score` | Риск, с которого отправляется alert. |
| `detection.alert-on-same-ip` | Отправлять alert при совпадении IP независимо от балла. |
| `detection.alert-on-similar-name` | Отправлять alert при похожем нике независимо от балла. |
| `geoip` | Страна по IP, кэш и лимиты HTTP-запросов. |

После изменения конфигурации используйте `/pnbans reload`.

## Команды и права

| Команда | Право |
| --- | --- |
| `/ban`, `/unban` | `pnbans.ban`, `pnbans.unban` |
| `/ipban`, `/unipban` | `pnbans.ban.ip`, `pnbans.unban.ip` |
| `/mute`, `/unmute` | `pnbans.mute`, `pnbans.unmute` |
| `/warn`, `/unwarn` | `pnbans.warn`, `pnbans.unwarn` |
| `/history` | `pnbans.history` |
| `/dupeip` | `pnbans.dupeip` |
| Staff-alert DupeIP | `pnbans.dupeip.alert` |
| Игнорировать DupeIP | `pnbans.dupeip.bypass` |
| Перезагрузка | `pnbans.admin` |

## Установка

1. Установите совместимую версию Paper 1.21+.
2. Поместите собранный JAR в папку `plugins`.
3. Запустите сервер и настройте `plugins/pnBans/config.yml` и `messages.yml`.
4. Для MySQL или MongoDB укажите подключение в `database` до первого запуска.

## Сборка из исходников

Нужны Java 21, Gradle и соседняя копия `pnLibrary`, потому что она подключается через `includeBuild('../pnLibrary')`.

```text
gradle build
```

В монорепозитории без системного Gradle можно использовать wrapper из `pnMarket`:

```text
..\pnMarket\gradlew.bat -p . build
```

Готовый файл появится в `build/libs/pnBans-1.0.0.jar`.

## Метрики

Плагин использует анонимную статистику bStats (ID `32598`) и отправляет только тип выбранной базы данных. Отключается глобально в `plugins/bStats/config.yml`.
